# v0.2.0 - Multi-agent runtime support

This release expands AI Builder beyond a single-agent workflow and makes the repository easier to use across Codex-style environments, Claude Code, and Gemini CLI.

## Highlights

- Added platform references for Codex, Claude Code, and Gemini CLI under `references/platforms/`
- Added a cross-runtime reference to keep one routing repository portable across multiple agent runtimes
- Added root adapter files `CLAUDE.md` and `GEMINI.md`
- Added safe settings templates in `.claude/settings.example.json` and `.gemini/settings.example.json`
- Added local-only overlay examples in `CLAUDE.local.example.md` and `GEMINI.local.example.md`
- Expanded installation guidance in `README.md`
- Strengthened repository validation for platform files, root adapters, and context imports
- Added a versioned `CHANGELOG.md`

## Why this matters

AI Builder can now be adopted more cleanly by teams that mix different coding agents while keeping:

- one shared routing model
- one shared builder library
- one shared security baseline

## Upgrade notes

For Claude Code:

- use `CLAUDE.md` as the thin project adapter
- copy `.claude/settings.example.json` to `.claude/settings.json` if you want a shared baseline
- copy `CLAUDE.local.example.md` to `CLAUDE.local.md` for personal local overrides

For Gemini CLI:

- use `GEMINI.md` as the thin project adapter, or configure `context.fileName` to include `AGENTS.md`
- copy `.gemini/settings.example.json` to `.gemini/settings.json` for a conservative workspace baseline
- use `GEMINI.local.example.md` as a personal overlay to import from your global or project `GEMINI.md`
