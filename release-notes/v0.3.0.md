# v0.3.0 - Six-platform portability

This release expands AI Builder to run across six distinct AI runtimes while keeping a single shared routing model.

## Highlights

- Added Antigravity adapter: `ANTIGRAVITY.md`, `.antigravity/rules/ai-builder.md`, `.antigravity/commands/ai-builder.md`
- Added ChatGPT Custom GPT integration: `references/platforms/chatgpt.md`, `dist/chatgpt-custom-gpt.md` paste-ready system prompt
- Added Claude Web Skills adapter: `references/platforms/claude-web.md`, `scripts/build_skill_bundle.py` producing `dist/ai-builder.zip`
- Added Claude Code native skill under `.claude/skills/ai-builder/SKILL.md` and slash command `.claude/commands/ai-builder.md`
- Rewrote the multi-runtime crosswalk to cover all six runtimes: Codex CLI, Claude Code, Claude Web, ChatGPT, Gemini CLI, and Antigravity
- Added platform-aware routing evaluation cases
- Strengthened repository validation with frontmatter limit checks for claude.ai Skills

## Why this matters

The same builder library and security baseline now works across every major AI coding agent without duplicating routing logic. Platform adapters are thin — the canonical instructions stay in `AGENTS.md` and `SKILL.md`.

## Installation

### Claude Web Skills

1. Download `dist/ai-builder.zip` from the release assets.
2. Go to claude.ai → Settings → Skills → Upload Skill.
3. Upload the zip — no further configuration needed.

### Claude Code

```bash
# Slash command (after cloning this repo into your Claude Code skills path)
/ai-builder <your request>
```

### ChatGPT Custom GPT

Copy `dist/chatgpt-custom-gpt.md` as the system prompt when creating a new Custom GPT.

### Codex CLI

```bash
$ai-builder <your request>
```

### Gemini CLI

The `GEMINI.md` root adapter imports `AGENTS.md` automatically when the repository is in context.

### Antigravity

The `.antigravity/` directory is read automatically when the repository is the active project.
