# v0.5.0 - Four new builders with patterns and recipes

This release adds Google Apps Script, TypeScript/Node.js, Bash/Shell, and React as first-class builders, each with a pattern library and at least one end-to-end recipe.

## Highlights

### New builders

- `google-apps-script-builder.md` — Google Workspace automation: Sheets macros, Docs generation, Gmail workflows, Drive management
- `typescript-node-builder.md` — CLI scripts, Node APIs, file-processing pipelines, Discord/Slack bots
- `bash-shell-builder.md` — Linux/macOS shell automation, CI/CD scripts, cron jobs, system administration
- `react-builder.md` — Internal tools and admin panels using Vite + TypeScript, no heavy framework

### New pattern libraries

One pattern file per builder with idioms, skeletons, and anti-patterns:
`google-apps-script-patterns.md`, `typescript-node-patterns.md`, `bash-shell-patterns.md`, `react-patterns.md`

### New recipes

- `sheets-dedup.md` — GAS dedup macro: batch read, backup tab, JS Set dedup, single setValues() write
- `node-api-csv.md` — TypeScript CLI: reads CSV, enriches via API, writes results CSV with dry-run flag
- `sql-query-to-excel.md` — Python: SQLite or DuckDB query → formatted Excel with frozen header
- `bash-bulk-archive.md` — Bash: moves PDFs to monthly folders with dry-run mode and CSV move log
- `react-kpi-dashboard.md` — Vite React: CSV drag-and-drop, KPI cards, Recharts line chart

### Extensions to existing builders

- `python-patterns.md` — New SQL patterns section: parameterized SQLite, DuckDB CSV-as-tables, pandas read_sql_query
- `html-js-patterns.md` — New Claude Artifacts section: single-file constraints, safe DOM, inline styles

## Why this matters

AI Builder now covers the most common vibe-coding scenarios beyond Office and Windows automation. The pattern + recipe layer gives agents enough depth to produce idiomatic, production-safe code in the first pass.

## Upgrade notes

Download `dist/ai-builder.zip` from the release assets and re-upload to replace the previous Claude Web Skill. All other runtimes pick up changes automatically on next context load.
