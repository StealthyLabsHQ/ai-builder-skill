# Builder Template

Use this template when adding a new file under `references/builders/`.

## Purpose

- State when the builder should be selected.

## Build Safely

- State the safety defaults for this implementation style.

## Structure The Solution

- State the preferred code shape, configuration style, and naming guidance.

## Tell The User How To Run It

- State where code should be saved or pasted.
- State the exact run pattern.
- State required prerequisites only when necessary.

## Validate And Protect Data

- State a quick validation path.
- State common failure modes.
- State backup or rollback guidance for destructive operations.
